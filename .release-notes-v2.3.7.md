### Bug Fixes

- **AppImage self-update no longer fails when AppImageLauncher is installed (#5):** electron-updater's built-in installer finishes an update by launching the new AppImage in a child process. AppImageLauncher intercepts that launch, the child exits early, and the update dies with a silent EPIPE - the app never actually updated and no error was shown. Nexus still uses electron-updater to detect and download updates (with sha512 verification), but the install step is now an atomic in-place file swap: the new AppImage is staged next to the old one and renamed over it, with no child process for AppImageLauncher to intercept. The new version takes effect the next time you launch the app. Windows installs are unchanged.

Thanks to @kraibse for pointing this out.
